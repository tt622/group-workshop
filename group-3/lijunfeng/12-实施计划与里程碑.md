## 1. 里程碑（S1～S4）

> **切分原则**：每刀切出一个**可演示的用户价值**。不是"接口写好了"，而是"能给业务看什么"。

**示例 — S1**：

**S1 研报概览与会话管理**：研报行业分类列表 + 会话 CRUD API + 前端概览页 → 满足 REQ-M1QA-001 部分 → 验证 TC-M01-020～025

| 阶段 | 交付要点（可演示） | 满足的 REQ/US | 验证 TC |
|------|-------------------|---------------|---------|
| **S1** 研报概览与会话管理 | 研报按申万二级行业分类展示 + 行业研报数量统计 + 会话 CRUD API + 前端概览页 | REQ-M1QA-001 部分 | TC-M01-020～025 |
| **S2** 研报解读核心 | 单篇研报AI解读（提取评级、目标价、核心观点）+ 公司研报历史解读列表 + 自然语言问答 | REQ-M1QA-001、REQ-M1QA-002 | TC-M01-001～010 |
| **S3** 数据对比与历史 | 多研报关键指标对比（同时间段）+ 公司间横向对比 + 历史问答记录查看 | REQ-M1QA-001 AC-001-03、REQ-M1QA-003 | TC-M01-030～040 |
| **S4** 发布准备 | 健康检查接口 + 操作留痕与安全控制 + 性能优化 + 全链路回归 | REQ-M1QA-004 | TC-M01-050～060 |

## 2. WBS 任务拆分

> **拆分原则**：按**工程职责**切（路由、存储、Agent、前端），不按功能切。一个 W 可跨多个 S 迭代推进。

| 任务ID | 任务名称 | 具体内容 | DoD（指向 Spec） | 推进阶段 |
|--------|----------|----------|------------------|----------|
| W1 | 路由实现 | Flask `agent_bp.py` 端点注册、参数校验（研报列表、解读、问答、对比等接口） | 对齐 `09` 端点与错误码 | S1 起贯穿 S4 |
| W2 | 存储层 | JSON 文件 CRUD：研报元数据、会话记录、问答历史、对比结果存储 | 对齐 `10` 数据模型 | S1～S2 |
| W3 | AI Agent 编排 | 研报解析提取（评级/目标价/核心观点）+ 自然语言问答 + 三级降级（CoPaw → 百炼 → Demo 模式） | 对齐 `08` 架构 Agent 层 | S2～S3 |
| W4 | 前端视图 | React + Vite：行业概览页、研报详情页、问答交互区、数据对比页，参照南方基金现有系统风格 | 对齐 `06` 功能规格 | S1 起贯穿 S4 |
| W5 | 数据对比引擎 | 多研报关键指标对比逻辑、同时间段约束校验、对比表自动生成 | 对齐 `06` 对比功能规格 | S3 |
| W6 | 测试门禁 | pytest 单元测试 + P0 用例覆盖 + 全链路回归 | 对齐 `13` 测试策略 | S1 起贯穿 S4 |

## 5. 依赖与风险

**依赖链**：

**S1 研报概览与会话管理** → **S2 研报解读核心**（必须先有研报分类和会话才能进行AI解读和问答）→ **S3 数据对比与历史**（必须先有解读结果才能对比）→ **S4 发布准备**

| 风险 | 影响 | 应对 |
|------|------|------|
| 上游 LLM 不可用 | 研报解读与问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| 数据源可理解性差（研报格式不统一） | 提取准确率下降，解析失败 | 多数据源融合获取 + 格式适配层 + 人工校验兜底 |
| 研报数据量大，每日开盘前生成压力 | S2 演示延期 | 增量解析 + 预处理调度优化 + 缓存机制 |
