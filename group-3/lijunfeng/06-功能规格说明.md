## 3. 行业概览页（对齐 US-001 AC-001-01）

> 用户可看到所有行业（按申万二级分类）的研报覆盖情况及每个行业的研报数量。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载行业列表 | 页面渲染 | `GET /reports` | 按申万二级行业分组展示，显示各行业研报数量 |
| 筛选行业 | 点击行业卡片 | `GET /reports?industry=xxx` | 展开该行业下的研报列表 |
| 日期筛选 | 选择时间段 | `GET /reports?date_from=&date_to=` | 按日期区间筛选研报 |
| 点击研报 | 点击研报标题 | 路由跳转 `/report/:id` | 进入研报详情页 |

### 3.1 行业卡片设计

```
┌───────────────────┐
│  🏭 电子          │
│  研报数量: 42      │
│  最新: 2026-04-13  │
│  ──────────────── │
│  覆盖公司: 15家    │
└───────────────────┘
```

## 4. 研报详情页（对齐 US-001 AC-001-02）

> 用户可看到具体某个公司的最新研报及历史研报解读。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载研报解读 | 页面渲染 | `GET /reports/<id>/reading` | 展示AI解读结果（评级、目标价、核心观点） |
| 触发新解读 | 点击"重新解读" | `POST /reports/<id>/reading` | loading → 展示最新解读结果 |
| 查看历史解读 | 切换"历史"tab | `GET /reports/<id>/reading` | 展示历史解读时间线列表 |

### 4.1 解读结果卡片

| 元素 | 类型 | 说明 |
|------|------|------|
| 评级标签 | `<span>` | 买入(红)/增持(橙)/中性(灰)/减持(绿)/卖出(深绿) |
| 目标价 | `<span>` | 数字展示，无目标价显示"—" |
| 核心观点 | `<ul>` | 有序列表，每条一行 |
| 财务预测 | `<table>` | 营收增速、PE 等关键指标表格 |
| 来源标签 | `<tag>` | copaw(蓝)/bailian(绿)/demo(灰) |

## 5. 数据对比页（对齐 US-001 AC-001-03）

> 用户可清晰看到历史数据对比以及与其他公司的对比，时间段可选但必须是同一时间段。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 选择对比研报 | 勾选研报（≥2 份） | — | 加入对比篮，按钮激活 |
| 选择时间段 | 日期选择器 | — | from/to 必须同时填写 |
| 发起对比 | 点击"开始对比" | `POST /reports/compare` | loading → 展示对比表格 |
| 查看对比表 | 对比完成 | — | 表格形式展示各研报各指标值 |

### 5.1 对比表格布局

```
┌──────────┬──────────┬──────────┬──────────┐
│  指标     │ 研报A     │ 研报B     │ 研报C    │
├──────────┼──────────┼──────────┼──────────┤
│ 评级      │ 买入      │ 增持      │ 中性     │
│ 目标价    │ 35.00    │ 32.50    │ 28.00    │
│ 营收增速  │ 15.2%    │ 12.8%    │ 10.5%    │
│ PE       │ 18.5x    │ 20.1x    │ 22.3x    │
└──────────┴──────────┴──────────┴──────────┘
```

## 6. Sidebar 会话管理（对齐 US-001）

> 会话管理侧栏在问答交互页展示，支持会话的增删查。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载列表 | 问答页渲染 | `GET /sessions` | 渲染会话列表 + 默认选中第一个 |
| 新建会话 | 点击"+ 新建" | `POST /sessions` | 插入列表头部 + 自动选中 |
| 删除会话 | 点击 × | `DELETE /sessions/<id>` | 确认弹窗 → 从列表移除 |
| 选中会话 | 点击会话项 | `GET /sessions/<id>/records` | 高亮选中 + 加载该会话的问答记录 |

## 7. 问答交互区（对齐 US-002）

### 7.1 Main 内容区（三态渲染）

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **A 空状态** | `currentSession === null` | "请创建或选择一个会话开始提问" |
| **B 引导态** | 有会话 && 无记录 | 展示常见问题示例网格，点击后自动填入输入框并提交 |
| **C 对话历史** | 有记录 | 按时间排序的问答卡片列表，每张卡片含提问、回答、来源标签、耗时 |

### 7.2 记录卡片中的来源标签逻辑

| `answer_source` | `llm_used` | 显示文本 | 样式 |
|-----------------|------------|----------|------|
| `copaw` | `true` | CoPaw | 蓝色标签 |
| `bailian` | `true` | 百炼 | 绿色标签 |
| `demo` | `false` | 离线演示 | 灰色标签 |

### 7.3 输入区域

| 元素 | 类型 | 行为 |
|------|------|------|
| query-input | `<textarea>` | 3 行，placeholder "请输入您的问题..." |
| send-btn | `<button>` | loading 时 disabled + 文案"发送中…"，query 非空才可点击 |
| clear-btn | `<button>` | 清空输入框 |

> 输入约束：query 非空且长度 1～500 字符（对齐 `05` US-002 AC）。

## 8. 历史记录页（对齐 US-003）

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 查看全部会话 | 页面渲染 | `GET /sessions` | 展示所有会话列表 |
| 查看会话记录 | 点击会话 | `GET /sessions/<id>/records` | 展开该会话的问答记录时间线 |

## 9. 错误处理

| 后端 `error.code` | HTTP | 前端展示 |
|--------------------|------|----------|
| `EMPTY_QUERY` | 400 | Toast 提示："请输入问题" |
| `INVALID_QUERY` | 400 | Toast 提示："问题过长，请控制在 500 字以内" |
| `MISSING_SESSION_ID` | 400 | Toast 提示："请先选择或创建一个会话" |
| `SESSION_NOT_FOUND` | 404 | Toast 提示："会话不存在，请刷新页面" |
| `REPORT_NOT_FOUND` | 404 | Toast 提示："研报不存在" |
| `INVALID_COMPARE_PARAMS` | 400 | Toast 提示："请选择至少 2 份研报并指定同一时间段" |
| `INTERNAL_ERROR` | 500 | Toast 提示："系统繁忙，请稍后重试" |

## 10. 状态管理

> React `useState` Hooks 管理的状态列表。

| State 变量 | 类型 | 初始值 | 用途 |
|------------|------|--------|------|
| `sessions` | `Session[]` | `[]` | 会话列表 |
| `currentSession` | `Session\|null` | `null` | 当前选中会话 |
| `records` | `QARecord[]` | `[]` | 当前会话的问答记录 |
| `query` | `string` | `""` | 输入框内容 |
| `loading` | `boolean` | `false` | 提交中状态 |
| `industries` | `Industry[]` | `[]` | 行业分类列表 |
| `currentReport` | `Report\|null` | `null` | 当前查看的研报 |
| `compareResults` | `CompareResult\|null` | `null` | 对比结果 |
| `capabilities` | `Caps\|null` | `null` | 能力探测结果 |
| `error` | `string\|null` | `null` | 错误消息 |
