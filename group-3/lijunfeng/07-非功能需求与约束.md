## 1. 性能需求

### 1.1 延迟 SLO

| 指标 | 目标值 | 度量方式 | 来源 |
|------|--------|----------|------|
| **问答端点 P95 延迟** | < **5000ms** | `response_time_ms` 字段 | `03` §4 |
| 会话管理端点延迟 | < 500ms | 服务端计时 | JSON I/O < 10ms |
| 研报列表查询延迟 | < 1000ms | 服务端计时 | 行业分类聚合 |
| 研报AI解读延迟 | < 30000ms | 服务端计时 | LLM 解析耗时较长 |
| 多研报对比延迟 | < 3000ms | 服务端计时 | 结构化数据对比 |
| 每日研报批量生成 | 开盘前（8:00）完成 | 定时任务监控 | `03` §4 成功标准 |

### 1.2 降级策略

> 对齐 `03` §5 风险：外部 LLM 不可用 → 三级降级。

```
CoPawAgent.ask(query, session_id)
  ├─[1] CoPaw 桥接 → answer_source='copaw', llm_used=True
  ├─[2] 百炼 DashScope → answer_source='bailian', llm_used=True
  └─[3] 离线演示兜底 → answer_source='demo', llm_used=False
```

| 约束 | 说明 |
|------|------|
| 三级降级**不可跳级** | 必须按 CoPaw → 百炼 → Demo 顺序尝试 |
| 降级**静默执行** | 不抛异常，通过 `answer_source` 标识 |
| Demo 模式**始终可用** | 无外部依赖，纯字符串拼接 |

### 1.3 吞吐量

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 并发用户数 | ≤ 20 | 一期内网部署，有限用户 |
| 自然语言问答命中率 | ≥ 80% | `03` §4 成功标准 |
| 全链路通过率 | "新建会话→提问→查看历史"3 分钟内完成 | `03` §4 成功标准 |

## 2. 可用性需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 服务可用性 | **99%**（内网环境） | 单实例 Flask，不设 HA |
| MTTR | < 5 分钟 | 重启 Flask 进程即可恢复 |
| 数据持久性 | JSON 文件定期备份 | 异常宕机后数据可恢复 |

### 2.1 故障模式与恢复

| 故障模式 | 影响范围 | 恢复方式 |
|----------|----------|----------|
| Flask 进程崩溃 | 全部 API 不可用 | 重启进程，< 5 分钟 |
| LLM 服务不可用 | 问答/解读降级 | 自动降级至 Demo 模式 |
| JSON 文件损坏 | 数据丢失 | 从备份恢复 |
| 研报源文件不可读 | 单篇解读失败 | 跳过并记录日志，不影响其他研报 |

## 3. 可观测性需求

### 3.1 链路追踪

| 项 | 规格 |
|----|------|
| traceId 格式 | `tr_{uuid.hex}`（32 位十六进制） |
| 注入位置 | 所有成功响应顶层 + 错误响应 `error.traceId` |
| 上游透传 | 优先复用请求头 `X-Trace-Id`，缺省时本地生成 |
| 操作留痕 | 所有写操作（创建/删除/解读/对比）记录 traceId + 时间戳 |

### 3.2 错误码体系

| HTTP | `error.code` | 触发条件 |
|------|--------------|----------|
| 400 | `EMPTY_QUERY` | query 为空/空白/null |
| 400 | `INVALID_QUERY` | query >500 字符 |
| 400 | `MISSING_SESSION_ID` | session_id 缺失 |
| 404 | `SESSION_NOT_FOUND` | session_id 不存在 |
| 404 | `REPORT_NOT_FOUND` | report_id 不存在 |
| 400 | `INVALID_COMPARE_PARAMS` | 对比参数不合法 |
| 500 | `UPSTREAM_ERROR` | Agent 编排内部异常 |
| 500 | `INTERNAL_ERROR` | 服务器内部错误 |

## 4. 安全需求

> 对齐 `03` §3 范围边界："操作留痕，安全控制"在范围内；"权限暂时不考虑"。

| 项 | 当前实现 | 说明 |
|----|----------|------|
| API 认证 | 无（开放访问） | 一期不做权限，内网部署 |
| CORS | 全开放 | 生产需白名单 |
| 速率限制 | 无 | 内网有限用户，暂不限速 |
| 数据安全 | 内网部署，数据不出域 | 用户访谈核心诉求（S-03） |

### 4.1 敏感数据保护

| 数据类型 | 保护方式 |
|----------|----------|
| LLM API Key | 环境变量存储（`.env`），禁止硬编码 |
| 用户查询 | 仅存储在本地 JSON，不传输到第三方（LLM 除外） |
| 研报原文 | 本地文件系统存储，禁止外传 |
| 解读结果 | 本地 JSON 存储，包含评级、目标价等敏感金融数据 |

## 5. 数据约束

### 5.1 输入约束

| 约束 | 规格 | 来源 |
|------|------|------|
| `query` 最小长度 | 1 字符 | `05` US-002 AC |
| `query` 最大长度 | 500 字符 | `05` US-002 AC |
| `session_id` 必填 | 非空字符串（UUID） | `09` POST /ask |
| 会话标题默认值 | "新会话"，首次问答自动重命名为 query 前 20 字 | `10` §9 业务逻辑 |
| 对比研报数量 | ≥ 2 份 | `05` US-001 AC-001-03 |
| 对比时间段 | from/to 必须同时提供，必须是同一时间段 | `05` US-001 AC-001-03 |

### 5.2 类型约束

| 字段 | 类型 | 约束 |
|------|------|------|
| `llm_used` | `boolean` | 必须为 True/False，不得为字符串 |
| `answer_source` | `string` | 枚举：copaw \| bailian \| demo |
| `rating` | `string` | 枚举：买入 \| 增持 \| 中性 \| 减持 \| 卖出 |
| `industry` | `string` | 必须为有效的申万二级行业分类 |
| `format` | `string` | 枚举：pdf \| html |
