## 1. 概念模型

```
Session（会话）1 ── * QARecord（问答记录）
   │                      │
   │  session_id (PK)     │  id (PK)
   │  title               │  session_id (FK → Session)
   │  query_count          │  query / answer / llm_used
   │  created_at           │  response_time_ms / answer_source
   └  updated_at           └  timestamp

ReportMeta（研报元数据）1 ── * ReportReading（研报解读）
   │                            │
   │  report_id (PK)            │  reading_id (PK)
   │  title                     │  report_id (FK → ReportMeta)
   │  industry (申万二级)        │  rating / target_price
   │  source (券商)              │  key_points / financial_forecast
   │  publish_date              │  reading_time
   │  format (PDF/HTML)         │
   └  company_name              └

CompareResult（对比结果）
   │
   │  compare_id (PK)
   │  report_ids (FK[] → ReportMeta)
   │  date_range {from, to}
   │  comparison_table
   └  created_at
```

> **核心关系**：
> - 一个 Session 可包含零到多条 QARecord，外键通过 `session_id` 逻辑引用
> - 一个 ReportMeta 可有零到多条 ReportReading，外键通过 `report_id` 逻辑引用
> - CompareResult 引用多个 ReportMeta

## 2. 存储引擎

| 项 | 说明 |
|----|------|
| 引擎 | JSON 文件（`json` 标准库） |
| 编码 | UTF-8，缩进 2 空格 |
| 读写模式 | RMW（全量读入 → 修改 → 全量写回） |
| 并发安全 | 无锁，不支持并发写入 |
| 文件布局 | `{DATA_DIR}/sessions.json` + `qa_records.json` + `reports_meta.json` + `report_readings.json` + `compare_results.json` |

## 3. 实体定义 — Session

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `session_id` | `string` (UUID) | 是 | **PK** | 会话唯一标识 |
| `title` | `string` | 是 | ≤23 字符 | 默认"新会话"，首次问答自动重命名 |
| `created_at` | `string` (ISO-8601) | 是 | UTC+Z | 创建时间 |
| `updated_at` | `string` (ISO-8601) | 是 | UTC+Z | 最后更新时间 |
| `query_count` | `integer` | 是 | ≥ 0 | 累计问答次数 |

## 4. 实体定义 — QARecord

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `id` | `string` | 是 | **PK**，`rec_{ts}` | 基于时间戳生成 |
| `session_id` | `string` | 是 | **FK** → Session | 所属会话 |
| `query` | `string` | 是 | 1–500 字符 | 用户提问原文 |
| `answer` | `string` | 是 | — | LLM/Demo 返回的答案 |
| `llm_used` | `boolean` | 是 | — | 是否使用真实 LLM |
| `model` | `string\|null` | 是 | — | 模型标识 |
| `response_time_ms` | `integer` | 是 | SLO < 5000 | 响应耗时（ms） |
| `answer_source` | `string\|null` | 否 | 枚举 | `copaw`\|`bailian`\|`demo` |
| `timestamp` | `string` (ISO-8601) | 是 | UTC+Z | 记录创建时间 |

## 5. 实体定义 — ReportMeta

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `report_id` | `string` (UUID) | 是 | **PK** | 研报唯一标识 |
| `title` | `string` | 是 | ≤200 字符 | 研报标题 |
| `industry` | `string` | 是 | 申万二级行业分类 | 所属行业 |
| `company_name` | `string` | 是 | — | 研报覆盖的公司名称 |
| `source` | `string` | 是 | — | 来源券商 |
| `publish_date` | `string` (ISO-8601) | 是 | — | 研报发布日期 |
| `format` | `string` | 是 | 枚举：`pdf`\|`html` | 研报文件格式 |
| `file_path` | `string` | 是 | — | 研报文件存储路径 |
| `created_at` | `string` (ISO-8601) | 是 | UTC+Z | 入库时间 |

## 6. 实体定义 — ReportReading

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `reading_id` | `string` (UUID) | 是 | **PK** | 解读记录唯一标识 |
| `report_id` | `string` | 是 | **FK** → ReportMeta | 所属研报 |
| `rating` | `string` | 是 | 枚举：买入/增持/中性/减持/卖出 | AI提取的评级 |
| `target_price` | `number\|null` | 否 | — | AI提取的目标价 |
| `key_points` | `array<string>` | 是 | — | 核心观点列表 |
| `financial_forecast` | `object\|null` | 否 | — | 财务预测（营收增速、PE 等） |
| `reading_time` | `string` (ISO-8601) | 是 | UTC+Z | 解读生成时间 |
| `llm_source` | `string` | 是 | 枚举 | `copaw`\|`bailian`\|`demo` |

## 7. 实体定义 — CompareResult

| 字段 | 类型 | 必填 | 约束 | 说明 |
|------|------|------|------|------|
| `compare_id` | `string` (UUID) | 是 | **PK** | 对比结果唯一标识 |
| `report_ids` | `array<string>` | 是 | ≥ 2 个 FK → ReportMeta | 参与对比的研报 |
| `date_range` | `object` | 是 | `{from, to}` ISO-8601 | 对比时间段（必须同一时间段） |
| `comparison_table` | `array<object>` | 是 | — | 对比表：`[{metric, values:{report_id: val}}]` |
| `created_at` | `string` (ISO-8601) | 是 | UTC+Z | 对比生成时间 |

## 9. 关键业务逻辑

| 逻辑 | 触发条件 | 行为 |
|------|----------|------|
| 首次问答自动命名 | `query_count` 从 0 → 1 | 标题 = query 前 20 字 + `...` |
| 级联删除 | 删除 Session | 同步删除所有关联 QARecord |
| 研报行业分类统计 | 查询研报列表 | 按申万二级行业分组并统计各行业研报数量 |
| 同时间段校验 | 触发多研报对比 | 校验所有参与对比的研报时间段一致，不一致则拒绝 |
| 每日自动生成 | 每日开盘前（8:00） | 批量触发当日新增研报的AI解读 |

```python
# 首次问答自动命名示例
if session["query_count"] == 1:
    session["title"] = query[:20] + ("..." if len(query) > 20 else "")
```

## 10. API 响应 DTO 概要

> 详细端点定义见 `09`。此处列出关键 DTO 与存储字段的映射关系。

| API 端点 | 核心 DTO 字段 | 额外注入 |

| `GET /reports/{id}/reading` | report_id, rating, target_price, key_points, financial_forecast | `traceId` |
| `POST /reports/compare` | compare_id, date_range, reports[], comparison_table[] | `traceId` |
