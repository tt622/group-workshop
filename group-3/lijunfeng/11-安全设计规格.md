## 1. 安全目标与合规要求

> 投研场景尤其关注数据隐私与内部信息防泄露（来源：`02` S-03 用户访谈 — "必须内部部署，数据安全第一"）。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务 | 南方基金内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 操作可追溯 | 所有关键操作留痕，支持审计回溯 | `03` §3 范围边界："操作留痕，安全控制" |

## 2. 认证与授权

> 对齐 `03` §3 范围边界："权限暂时不考虑，需求没提"。一期不做认证，但预留扩展接口。

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 一期：无（开放访问，内网部署）；二期：对接南方基金内部 SSO |
| Token 有效期 | 一期不适用；二期：Access Token 30min, Refresh Token 7d |
| 密码策略 | 一期不适用；二期：对接统一身份认证，遵循公司密码策略 |
| 多因素认证 | 一期不适用；二期：按公司安全等级要求接入 |

### 2.2 角色与权限矩阵

> 一期所有用户角色等同，无权限区分。二期规划如下：

| 角色 | 研报上传 | 研报阅读 | 问答交互 | 对比分析 | 系统管理 |
|------|----------|----------|----------|----------|----------|
| 投研分析师 | ✗ | ✓ | ✓ | ✓ | ✗ |
| 投研总监 | ✗ | ✓ | ✓ | ✓ | ✗ |
| 数据管理员 | ✓ | ✓ | ✗ | ✗ | ✓ |
| 系统管理员 | ✓ | ✓ | ✓ | ✓ | ✓ |

## 3. 数据安全

> 投研场景核心关注点：研报涉及未公开信息，问答记录包含投资决策线索。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/HTML 研报文件 | 本地存储，禁止外传，访问审计 |
| AI解读结果 | **机密** | 评级、目标价、核心观点 | 本地 JSON 存储，操作留痕 |
| 问答记录 | **内部** | 用户提问与AI回答 | 本地存储，定期清理 |
| 会话数据 | **内部** | 会话标题、创建时间 | 本地 JSON 存储 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 当前（一期） | 生产改进（二期） |
|------|-------------|-----------------|
| 传输加密 | HTTP（内网） | TLS 1.2+ 强制 HTTPS |
| 存储加密 | 明文 JSON | AES-256 落盘加密 |
| 密钥管理 | 环境变量 `.env` | KMS / Vault，定期轮转 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 当前实现（一期） | 生产改进（二期） |
|----------|-----------------|-----------------|
| 输入校验 | 服务端白名单校验（query 1-500 字符、UUID 格式等） | 增加 WAF 规则 |
| SQL 注入防护 | 不适用（JSON 文件存储） | P2 迁移数据库后使用参数化查询 |
| XSS 防护 | React 默认转义 | 增加 CSP 头，研报 HTML 渲染沙箱化 |
| CSRF 防护 | 无（REST API，无 Cookie） | 二期 SSO 后增加 CSRF Token |
| 速率限制 | 无（内网有限用户） | 100 req/min/user |
| 文件上传校验 | 不适用（一期不做研报上传） | 二期：类型白名单（PDF/HTML）+ 大小限制 50MB |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署 CoPaw / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤特殊指令 | 注入测试用例全部拦截 |
| 模型幻觉 / 错误信息 | 回答标注 `answer_source`，研报解读标注原文引用 | 所有回答含来源标识 |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配内部代码、密码等敏感信息并脱敏 | 脱敏规则覆盖率 100% |
| Demo 模式数据真实性 | Demo 模式明确标识为离线演示，使用模拟数据 | UI 显示灰色"离线演示"标签 |

## 6. 审计与日志

> 对齐 `03` §3 范围边界："操作留痕，安全控制"。

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 研报解读触发 | traceId、report_id、操作时间、llm_source | 365 天 | — |
| 问答提交 | traceId、session_id、query 摘要、answer_source | 365 天 | — |
| 会话创建/删除 | traceId、session_id、操作类型、时间戳 | 180 天 | 批量删除告警 |
| 数据对比 | traceId、report_ids、对比时间段 | 180 天 | — |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 关键配置变更即时通知 |
| 异常/错误 | traceId、error.code、堆栈摘要 | 90 天 | 5xx 错误 ≥ 10 次/分钟告警 |

## 7. 安全威胁清单（STRIDE 简表）

| 类型 | 威胁 | 风险等级 | 应对措施 |
|------|------|----------|----------|
| **S**（伪造） | 伪造用户身份访问研报 | 中（一期内网低风险） | 一期内网隔离；二期 SSO + Token 校验 |
| **T**（篡改） | 研报内容在传输中被篡改 | 中 | 一期内网 HTTP；二期 TLS + 文件哈希校验 |
| **R**（抵赖） | 用户否认曾进行某操作 | 中 | traceId 全链路留痕，操作日志不可篡改 |
| **I**（泄露） | 内部研报数据通过AI模型泄露 | **高** | 本地部署 + 内网百炼 + 输出过滤 |
| **D**（拒绝服务） | 大量请求导致服务不可用 | 低（内网有限用户） | 三级降级策略 + 二期速率限制 |
| **E**（提权） | 普通用户访问管理功能 | 低（一期无权限区分） | 二期 RBAC 角色权限矩阵 |

## 8. 部署安全

- [x] 全部服务**内网部署**，数据不出域（核心安全要求）
- [x] 敏感配置（API Key）通过环境变量 `.env` 注入，不入代码仓库
- [x] `.env` 文件加入 `.gitignore`
- [ ] 生产/开发环境**严格隔离**，数据不互通（二期）
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] 研报文件存储目录权限设置为仅应用进程可读写
- [ ] 日志文件定期归档，敏感信息脱敏后存储
