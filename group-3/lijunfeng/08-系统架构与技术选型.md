## 1. 架构总览

### 1.1 系统上下文

```
┌─────────────────────────────────┐
│       用户浏览器 React SPA       │
│       http://localhost:5173     │
│  （南方基金欧拉平台风格）         │
└──────────────┬──────────────────┘
               │ HTTP  /api/v1/agent/*
               ▼
┌─────────────────────────────────┐
│     Flask Backend (WSGI)        │
│     http://localhost:5000       │
│  ┌────────────────────────────┐ │
│  │  agent_bp — API 端点       │ │
│  │    ↓          ↓         ↓  │ │
│  │ ReportParser CoPawAgent   │ │
│  │ CompareEngine Storage     │ │
│  └────┬──────────┬────────────┘ │
│       ▼          ▼              │
│  ┌────────┐ ┌──────────┐       │
│  │ LLM 层 │ │ 文件系统  │       │
│  │CoPaw   │ │ ./data/  │       │
│  │百炼    │ │ *.json   │       │
│  │Demo    │ └──────────┘       │
│  └────────┘                    │
│       ▼                         │
│  ┌──────────────┐              │
│  │ 外部数据源    │              │
│  │ 研报PDF/HTML  │              │
│  └──────────────┘              │
└─────────────────────────────────┘
```

### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 前端独立开发，参照南方基金现有欧拉平台风格 |
| **单体部署** | 前后端各一个进程 | 一期不引入微服务复杂度，快速交付可演示版本 |
| **分层架构** | Route → Agent/Parser → Storage | 替换存储引擎或 LLM 提供商时上层零改动 |
| **本地部署** | 全部服务内网运行 | 数据安全第一，数据不出域（用户访谈 S-03 核心诉求） |

## 2. 后端分层结构

> 每层写清"允许做什么、禁止做什么"。

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `agent_bp.py` | 参数校验、HTTP 状态码、traceId 注入 | 直接读写文件、调用 LLM |
| **Agent** | `agent.py` | 降级编排、问答结果组装 | 感知 HTTP 请求/响应 |
| **Parser** | `report_parser.py` | 研报解析、关键信息提取（评级/目标价/核心观点） | 操作本地存储 |
| **Compare** | `compare_engine.py` | 多研报指标对比、同时间段校验 | 调用外部 API |
| **Provider** | `copaw_bridge.py` / `bailian_qa.py` | HTTP 调用外部 LLM API | 操作本地存储 |
| **Storage** | `storage.py` | JSON 文件 CRUD、数据校验 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）≥ 18 | 通用性高，团队技术栈匹配 |
| 构建工具 | Vite ≥ 5 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` | 页面状态简单，无需引入 Redux |
| HTTP 客户端 | `fetch` API | 浏览器原生，减少依赖 |
| 样式方案 | CSS Modules / 南方基金组件库 | 参照欧拉平台现有风格，保持 UI 一致性 |

| 页面 | 路由 | 功能 | 对应 REQ |

## 4. 三级降级编排

> 对齐 `07` 降级策略（来源：03 §5 风险）。

```
ask(query, session_id)
  ├─[1] CoPaw  → {source:"copaw",  llm:true}
  ├─[2] 百炼   → {source:"bailian",llm:true}
  └─[3] Demo   → {source:"demo",   llm:false}
```

| 提供商 | 文件 | 配置检测 | 超时 | 错误处理 |
|--------|------|----------|------|----------|
| **CoPaw** | `copaw_bridge.py` | `IRA_COPAW_*_URL` 非空 | 20s | 返回 None 静默降级 |
| **百炼** | `bailian_qa.py` | `DASHSCOPE_API_KEY` 非空 | 120s | 区分多类错误码 |
| **Demo** | `agent.py` | 始终可用 | 0s | 纯字符串拼接 |

## 5. 数据存储架构

### 5.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | JSON 文件 | — | 一期快速交付，适合演示 |
| P1 | SQLite | 仅 Storage 层重写 | 支持基本并发、查询性能提升 |
| P2 | PostgreSQL / MySQL | Storage 层 + 部署配置 | 生产级，支持高并发和数据备份 |

### 5.2 文件布局

```
./data/
  ├── sessions.json         # 会话数据
  ├── qa_records.json        # 问答记录
  ├── reports_meta.json      # 研报元数据（行业分类、评级等）
  ├── report_readings.json   # 研报AI解读结果
  └── compare_results.json   # 数据对比结果缓存
```

## 7. ADR（架构决策记录）

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 团队 Python 基础即可上手，学习曲线低 |
| ADR-002 | JSON 文件而非数据库 | 一期快速交付，无需额外部署数据库，Storage 层抽象后可平滑升级 |
| ADR-003 | 本地部署而非云端 | 用户访谈明确要求数据不出域（S-03：必须内部部署，数据安全第一） |
| ADR-004 | 研报解析独立为 Parser 层 | 研报格式不统一（PDF/HTML），独立封装便于适配多种数据源格式 |

## 8. 安全架构

> 对齐 `03` §5 风险与范围边界中的安全控制要求。

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT + 统一登录网关 |
| CORS | 全开放 | 欧拉平台域名白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 数据安全 | 内网部署，数据不出域 | 数据加密存储 + 审计日志 |
| 操作留痕 | traceId 链路追踪 | 全量操作日志 + 行为审计 |
