## 1. 愿景与目标

> 把 `03` §1 的痛点翻译成正面目标。格式："构建 XXX，帮助用户在 XXX 内完成 XXX"。

构建统一的投研问答助手，帮助投研分析师在一个入口内完成研报查询、观点整合、预测验证等全流程投研分析工作。

| 03 痛点 | 04 目标 |
|---------|---------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 统一会话问答入口，汇聚多源信息，一站式查询 |
| 总结单个研报观点，整理多个研报的共同观点和差异化观点 | 智能观点整合：自动提取、分类、对比多份研报观点，生成结构化摘要 |
| 对研报中前瞻和预测性观点，结合大盘数据进行验证 | 观点回测验证：将预测性观点与实际大盘数据关联，辅助判断观点可靠性 |

## 2. 目标用户

> 把 `03` §2 的一句话展开成具体角色。思考：谁会用？用来做什么？多常用？

| 角色 | 使用目标 | 频率 |
|------|----------|------|
| 投研分析师 | 快速查询投研观点与市场解释，整合多份研报观点 | 每天多次 |
| 基金经理 | 查看观点摘要与验证结果，辅助投资决策 | 每天 1~2 次 |
| 研究助理 | 批量查阅研报、追踪历史问答记录，积累研究素材 | 每天多次 |

## 3. 核心业务场景

> 把 `03` §2 提案概要拆成 4~5 条用户路径。每条场景 = 一条用户操作动线。

**SC-01 新建会话并提问**：用户点击"新会话" → 输入问题 → 获得回答（含来源标识）→ 回答自动落库

| 编号 | 场景名 | 用户路径 |
|------|--------|----------|
| SC-01 | 新建会话并提问 | 用户点击"新会话" → 输入问题 → 获得回答（含来源标识）→ 回答自动落库 |
| SC-02 | 观点整合与摘要 | 用户提问"XX行业近期研报有哪些共同观点？" → 系统检索多份研报 → 提取核心观点 → 分类整合 → 生成结构化摘要返回 |
| SC-03 | 观点回测与验证 | 用户选择某个预测性观点 → 系统自动关联大盘/行业指数数据 → 对比验证 → 返回验证结论与趋势图 |
| SC-04 | 查看历史问答记录 | 用户点击侧栏历史会话 → 加载该会话下所有问答记录 → 支持上下文回看 |
| SC-05 | 会话管理 | 用户创建/删除会话 → 会话列表实时更新 → 删除时级联清除关联记录 |

## 4. 产品规则

> 把 `03` §4 成功标准 + §3 范围约束展开成可落地的规则。

| 编号 | 规则 | 来源 |
|------|------|------|
| R-01 | 每条提问必须绑定 `session_id` | 03 §2 "会话式问答" |
| R-02 | `query` 长度 1～500 字符，为空或超长返回明确错误 | 03 §4 "P0 全部可测" |
| R-03 | 回答必须携带 `answer`、`llm_used`(bool)、`model`、`response_time_ms`、`answer_source` 字段 | 03 §4 可追溯要求 |
| R-04 | 降级策略：CoPaw → 百炼 → Demo 模式，三级不可跳级，降级静默执行通过 `answer_source` 标识 | 03 §5 风险应对 |
| R-05 | 会话状态规则：新建会话默认标题"新会话"；首次问答后自动以 query 前 20 字重命名 | 03 §3 会话管理 |

## 5. 能力与优先级

> `03` §3 范围表写了"做什么"，04 按 P0/P1 排优先级。P0 = 必须实现，阻塞发布；P1 = 应当实现，不阻塞首版。

| 子能力 | 优先级 | 对应场景 |
|--------|--------|----------|
| 会话管理（列表 / 新建 / 删除） | **P0** | SC-01, SC-05 |
| 问答提交与三级降级 | **P0** | SC-01 |
| 观点整合与分类摘要 | **P0** | SC-02 |
| 观点回测与大盘数据验证 | **P0** | SC-03 |
| 历史问答记录查看 | **P1** | SC-04 |

## 6. 验收与成功标准（PRD 层）

> 从 `03` §4 可度量标准出发，加上演示条件。

| 标准 | 度量 |
|------|------|
| 可演示 | 种子数据 ≥ 10 条下，主路径 + 历史 + 能力状态 **3 分钟内**演示完毕 |
| 离线可用 | 未配置密钥时系统仍返回离线演示回复（`answer_source=demo`） |
| 观点整合 | 输入 2 份以上研报后，系统能输出共同观点与差异观点的结构化摘要 |
| 观点验证 | 对预测性观点，能关联至少一项大盘指标数据进行趋势对比验证 |

## 7. 与 05 用户故事的关系

> **04 定"做什么、多重要"；05 定"谁要什么、怎么验收"。** 填写完本文后，下一步进入 `05` 将场景拆成 US + AC。

| 本文（PRD） | 作用 | 05 中落地 |
|-------------|------|-----------|
| §3 核心场景 | 一条叙事场景 → 一条或多条 US | SC-01 → US-001 + US-002 |
| §4 产品规则 | 字段/状态/错误口径 | 拆入各 US 的 AC |
| §5 能力优先级 | P0 / P1 | 与各 US 优先级一致 |
