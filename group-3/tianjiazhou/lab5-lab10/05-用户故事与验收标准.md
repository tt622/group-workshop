> **本文只做一件事**：按用户能理解的子场景拆成多条 **User Story**，并为每条写清 **REQ-ID**、**验收标准（AC）**。

## 字段说明 — 每条 UserStory 必须包含以下字段

| 字段 | 填写说明 | 数据来源 ← | 流向下游 → |
|------|----------|------------|-----------|
| **需求编号** | 唯一编号，如 REQ-M1QA-001 | ← `02` 采集编号 | → `14` 追踪矩阵首列 |
| **故事** | 作为【角色】，我希望【行为】，以便【价值】 | ← `04` PRD 场景拆解 | → `06` FSD 功能点 |
| **优先级** | P0（必须）/ P1（应该）/ P2（可选） | ← `03` §4 成功标准 | → `12` 里程碑排期 |
| **来源** | 对应 03/04 哪一条需求 | ← `03` 范围 · `04` 场景 | |
| **前置条件** | 触发该故事的前提状态 | ← `04` 用户流程 | → `13` 测试前置 |
| **功能对齐** | 对应功能规格 + 非功能约束 | | → `06` FSD · `07` NFR |
| **设计对齐** | 对应架构、接口、数据、安全 | | → `08` 架构 · `09` API · `10` Data · `11` 安全 |
| **验证对齐** | 对应测试用例 + 追踪矩阵行 | | → `13` TC · `14` 追踪矩阵行 |
| **验收标准** | AC，每条可独立验证 | ← 故事 + `07` NFR | → `13` 测试用例依据 |

## US-001（P0）会话管理

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-001 |
| **故事** | 作为投研分析师，我希望能创建、查看和删除会话，以便按主题组织我的问答记录 |
| **优先级** | P0 |
| **来源** | ← 03 §3 范围"会话管理：创建、列表、删除" / 04 §3 SC-05 |
| **前置条件** | 用户已打开系统页面 |
| **功能对齐** | → 06 §3 Sidebar 会话管理 · 07 §2 可用性需求 |
| **设计对齐** | → 08 §2 后端分层 · 09 POST/GET/DELETE /sessions · 10 Session 表 · 11 §4 接口安全 |
| **验证对齐** | → 13 TC-M01-020~025 · 14 第 6 行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-001-01 | 点击"新建会话"后，`POST /sessions` 返回 201，侧栏新增一条会话，标题默认"新会话" |
| AC-001-02 | 页面加载时 `GET /sessions` 返回 200，侧栏渲染全部会话列表并默认选中第一个 |
| AC-001-03 | 点击会话删除按钮后弹出确认框，确认后 `DELETE /sessions/<id>` 返回 200，会话及关联记录被移除 |

## US-002（P0）问答提交

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M1QA-002 |
| **故事** | 作为投研分析师，我希望在会话中输入问题并获得AI回答，以便快速获取投研观点与市场分析 |
| **优先级** | P0 |
| **来源** | ← 03 §3 范围"会话管理" / 04 §3 SC-01 |
| **前置条件** | 已创建并选中一个会话 |
| **功能对齐** | → 06 §5 输入区域 · 07 §1 性能需求（P95 < 5000ms） |
| **设计对齐** | → 08 §4 三级降级编排 · 09 POST /ask · 10 QARecord 表 · 11 §5 LLM 安全 |
| **验证对齐** | → 13 TC-M01-001~004 · 14 第 1~3 行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-002-01 | 输入有效 query（1~500字符）+ session_id，`POST /ask` 返回 200，包含 answer、llm_used、model、response_time_ms、answer_source 字段 |
| AC-002-02 | query 为空时返回 400 + `EMPTY_QUERY`；query 超 500 字符时返回 400 + `INVALID_QUERY` |
| AC-002-03 | 三级降级正常工作：CoPaw 不可用时自动降级百炼，百炼不可用时降级 Demo，`answer_source` 正确标识来源 |
