## 1. 里程碑（S1～S4）

> **切分原则**：每刀切出一个**可演示的用户价值**。不是"接口写好了"，而是"能给业务看什么"。

| 阶段 | 交付要点（可演示） | 满足的 REQ/US | 验证 TC |
|------|-------------------|---------------|---------|
| **S1** 会话管理 | 会话 CRUD API + 前端侧栏：可创建/删除/切换会话 | REQ-001 (US-001) | TC-M01-020～025 |
| **S2** 问答核心 | 问答提交 API + 三级降级 + 前端提问区：可在会话中提问并获得 AI 回答 | REQ-002 (US-002) | TC-M01-001～004 |
| **S3** 观点分析 | 观点整合摘要 + 观点回测验证 + 历史记录查看 | REQ-003～005 (US-003～005) | TC-M01-030～031, 050～057 |

> **拆分原则**：按**工程职责**切（路由、存储、Agent、前端），不按功能切。一个 W 可跨多个 S 迭代推进。

| 任务ID | 任务名称 | 具体内容 | DoD（指向 Spec） | 推进阶段 |
|--------|----------|----------|------------------|----------|
| W1 | 路由实现 | `agent_bp.py` 端点注册、参数校验、traceId 注入 | 对齐 `09` 全部 6 个端点与错误码 | S1 起贯穿 S4 |
| W2 | 存储层 | `storage.py` JSON CRUD：Session + QARecord | 对齐 `10` 数据模型与方法清单 | S1～S2 |
| W3 | Agent 编排 | `agent.py` 三级降级链 + CoPaw/百炼/Demo 提供商 | 对齐 `08` §4 降级编排 | S2～S3 |
| W4 | 前端视图 | React SPA：Header + Sidebar + Main + Input | 对齐 `06` 全部 UI 规格 | S1 起贯穿 S4 |
| W5 | 观点分析 | 观点整合、分类、回测验证逻辑 | 对齐 `05` US-004/005 验收标准 | S3 |

## 5. 依赖与风险

**依赖链**：

**S1 会话管理** → **S2 问答核心**（必须先有会话才能提问）→ **S3 观点分析与历史**（依赖问答数据积累）→ **S4 收口发布**

| 风险 | 影响 | 应对 |
|------|------|------|
| 上游 LLM 不可用 | 问答降级，观点分析无法使用真实模型 | Demo 模式兜底 + 种子数据演示 |
| 大盘数据接口不稳定 | 观点回测验证功能受限 | 预置历史数据作为兜底 |
| 研报格式不统一 | 观点提取准确率下降 | 标准化预处理流程 + 人工校验 |
