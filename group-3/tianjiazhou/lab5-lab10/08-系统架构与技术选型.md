## 1. 架构总览

### 1.1 系统上下文

```
┌─────────────────────────────────┐
│       用户浏览器 React SPA       │
│       http://localhost:5173     │
└──────────────┬──────────────────┘
               │ HTTP  /api/v1/agent/*
               ▼
┌─────────────────────────────────┐
│     Flask Backend (WSGI)        │
│     http://localhost:5000       │
│  ┌────────────────────────────┐ │
│  │  agent_bp — 6 个 API 端点  │ │
│  │    ↓          ↓         ↓  │ │
│  │ CoPawAgent  Storage  _ok() │ │
│  └────┬──────────┬────────────┘ │
│       ▼          ▼              │
│  ┌────────┐ ┌──────────┐       │
│  │ LLM 层 │ │ 文件系统  │       │
│  │CoPaw   │ │ ./data/  │       │
│  │百炼    │ │ *.json   │       │
│  │Demo    │ └──────────┘       │
│  └────────┘                    │
└─────────────────────────────────┘
```

### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 教学清晰度：前端独立开发服务器 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度 |
| **分层架构** | Route → Agent → Storage | 替换存储引擎时上层零改动 |
| **降级优先** | 三级降级链始终可用 | 外部依赖不可用时不影响核心功能演示 |

## 2. 后端分层结构

> 每层写清"允许做什么、禁止做什么"。

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `agent_bp.py` | 参数校验、HTTP 状态码、traceId | 直接读写文件、调用 LLM |
| **Agent** | `agent.py` | 降级编排、结果组装 | 感知 HTTP 请求/响应 |
| **Provider** | `copaw_bridge.py` / `bailian_qa.py` | HTTP 调用外部 API | 操作本地存储 |
| **Storage** | `storage.py` | 文件 CRUD、数据校验 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）≥ 18 | 教学通用性最高 |
| 构建工具 | Vite ≥ 5 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` | 8 个 state 变量足够，不引入 Redux |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| 样式方案 | CSS Modules / 内联样式 | 避免全局样式污染，教学简洁 |

### 3.1 组件结构

```
App
├── Header（标题 + 能力芯片）
├── Sidebar（会话列表 + 新建/删除）
├── MainContent（三态渲染）
│   ├── EmptyState（空状态引导）
│   ├── SuggestedQuestions（常见问题网格）
│   └── ChatHistory（对话历史卡片列表）
│       └── MessageCard（单条问答卡片）
├── InputArea（输入框 + 发送按钮）
└── DeleteConfirmModal（删除确认弹窗）
```

## 4. 三级降级编排

> 对齐 `07` §1.2 降级策略。

```
ask(query, session_id)
  ├─[1] CoPaw  → {source:"copaw",  llm:true}
  ├─[2] 百炼   → {source:"bailian",llm:true}
  └─[3] Demo   → {source:"demo",   llm:false}
```

| 提供商 | 文件 | 配置检测 | 超时 | 错误处理 |
|--------|------|----------|------|----------|
| **CoPaw** | `copaw_bridge.py` | `IRA_COPAW_*_URL` 非空 | 20s | 返回 None 静默降级 |
| **百炼** | `bailian_qa.py` | `DASHSCOPE_API_KEY` 非空 | 120s | 区分多类错误码 |
| **Demo** | `agent.py` | 始终可用 | 0s | 纯字符串拼接 |

## 5. 数据存储架构

### 5.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | JSON 文件 | — | 适合教学，零依赖 |
| P1 | SQLite | 仅 Storage 层 | 支持并发读、索引查询 |
| P2 | PostgreSQL | Storage 层 + 部署配置 | 生产级，支持事务和并发写 |

### 5.2 文件布局

```
./data/
├── sessions.json       # 会话列表
└── qa_records.json     # 问答记录列表
```

## 6. 部署架构

| 服务 | 端口 | 启动命令 |
|------|------|----------|
| Flask Backend | **5000** | `python -m flask --app wsgi run --port 5000` |
| Vite Frontend | **5173** | `npm run dev` |

### 6.1 环境变量

| 变量 | 必填 | 说明 |
|------|------|------|
| `IRA_COPAW_ASK_URL` | 否 | CoPaw 问答接口地址 |
| `IRA_COPAW_SESSION_URL` | 否 | CoPaw 会话接口地址 |
| `DASHSCOPE_API_KEY` | 否 | 百炼 API 密钥 |
| `DATA_DIR` | 否 | 数据目录路径，默认 `./data` |

## 7. ADR（架构决策记录）

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线低，无需 async/await，学员 Python 基础即可上手 |
| ADR-002 | JSON 文件而非 SQLite | 零依赖启动，肉眼可读调试，教学透明度高 |
| ADR-003 | 不拆前端组件库 | 教学场景组件数少（< 10），单文件 App 即可维护 |
| ADR-004 | fetch 而非 axios | 减少依赖，浏览器原生 API 足够覆盖所有场景 |

## 8. 安全架构

> 对齐 `07` §4 安全需求。当前教学版与生产需求的差距。

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT Token |
| CORS | 全开放 | 白名单（仅允许前端域名） |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 输入校验 | 长度 + 非空校验 | 增加 XSS/注入过滤 |
| 速率限制 | 无 | 令牌桶限流（100 req/min） |
