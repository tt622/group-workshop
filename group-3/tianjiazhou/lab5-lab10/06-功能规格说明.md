## 1. 页面总体布局

> SPA 单页应用，分为 Header + Sidebar + Main + Input Area。

```
┌──────────────────────────────────────────────┐
│  header: 投研问答助手 + [能力状态芯片]          │
├────────────┬─────────────────────────────────┤
│  sidebar   │  main-content（三态渲染）         │
│  历史会话   │  A) 空状态: "请创建或选择会话"     │
│  [+新建]   │  B) 常见问题网格（引导提问）       │
│  session1  │  C) 对话历史列表                  │
│  session2  │     ┌─ 用户提问卡片               │
│  session3  │     └─ AI回答卡片(含来源标签)      │
│            │                                  │
│            │  input-area: textarea + 发送按钮   │
└────────────┴─────────────────────────────────┘
```

## 2. Header 区域

> 描述 Header 包含哪些元素。重点关注能力状态芯片（对齐 `05` US-006）。

### 2.1 能力状态芯片（Capability Chips）

| 条件 | 芯片显示 | 说明 |
|------|----------|------|
| `caps.copaw_configured = true` | CoPaw 桥接 | CoPaw 已配置，绿色标签 |
| `caps.bailian_configured = true` | 百炼 · {model} | 百炼已配置 + 模型名，蓝色标签 |
| `caps.bailian_configured = false` 且 `caps.copaw_configured = false` | 离线演示 | 降级到离线模式，灰色标签 |

### 2.2 Header 元素

| 元素 | 说明 |
|------|------|
| 应用标题 | "投研问答助手"，左对齐 |
| 能力芯片组 | 右对齐，页面加载时调用 `GET /capabilities` 获取状态 |

## 3. Sidebar 会话管理

> 对齐 `05` US-001。描述会话列表的增删查交互。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载列表 | 页面渲染 | `GET /sessions` | 渲染会话列表 + 默认选中第一个 |
| 新建会话 | 点击"+ 新建" | `POST /sessions` | 插入列表头部 + 自动选中 + 标题"新会话" |
| 删除会话 | 点击 × | `DELETE /sessions/<id>` | 确认弹窗 → 从列表移除 → 若删除当前选中则自动选中下一个 |
| 选中会话 | 点击会话条目 | `GET /sessions/<id>/records` | 高亮当前会话 → 加载该会话问答记录到 Main 区 |

### 3.1 会话条目显示

| 元素 | 说明 |
|------|------|
| 标题 | 显示 `title` 字段，最多 20 字截断 + `...` |
| 问答数 | 显示 `query_count`，如"3 条问答" |
| 删除按钮 | hover 时显示 × 按钮 |

## 4. Main 内容区（三态渲染）

> 对齐 `05` US-002（问答提交）、US-003（历史记录）、US-004（观点整合）、US-005（观点验证）。

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **A 空状态** | `currentSession === null` | 居中提示"请创建或选择一个会话开始"，配合引导图标 |
| **B 常见问题** | 有会话 && 无记录 | 显示 3×2 常见问题卡片网格，如"近期医药行业有哪些共识观点？""某基金经理的预测准确率如何？"，点击自动填入输入框并发送 |
| **C 对话历史** | 有记录 | 按时间正序渲染问答卡片列表，新消息自动滚到底部 |

### 4.1 记录卡片结构

每条问答记录包含：

| 元素 | 说明 |
|------|------|
| 用户提问 | 右对齐气泡，显示 `query` 文本 |
| AI 回答 | 左对齐气泡，显示 `answer` 文本（支持 Markdown 渲染） |
| 来源标签 | 回答气泡右下角，根据 `answer_source` 和 `llm_used` 显示 |
| 响应耗时 | 回答气泡右下角，显示 `response_time_ms` ms |
| 时间戳 | 卡片底部居中，显示 `timestamp` |

### 4.2 来源标签逻辑

| `answer_source` | `llm_used` | 显示文本 | 样式 |
|-----------------|------------|----------|------|
| `copaw` | `true` | CoPaw | 绿色标签 |
| `bailian` | `true` | 百炼 | 蓝色标签 |
| `demo` | `false` | 离线演示 | 灰色标签 |

## 5. 输入区域

> 对齐 `05` US-002 AC（query 非空且长度 1~500）。

| 元素 | 类型 | 行为 |
|------|------|------|
| query-input | `<textarea>` | 3 行高度，placeholder "请输入您的投研问题..."，支持 Enter 发送 / Shift+Enter 换行 |
| send-btn | `<button>` | 发送中 disabled + 文案"分析中…" + loading 动画；无会话选中时 disabled |
| char-counter | `<span>` | 实时显示字符数 `N/500`，超 450 变橙色警告，超 500 变红禁止发送 |

## 6. 错误处理

| 后端 `error.code` | HTTP | 前端展示 |
|--------------------|------|----------|
| `EMPTY_QUERY` | 400 | 输入框下方红色提示："请输入问题" |
| `INVALID_QUERY` | 400 | 输入框下方红色提示："问题过长（最多 500 字符）" |
| `UPSTREAM_ERROR` | 500 | Toast 弹出："服务暂时不可用，请稍后重试"，自动 3 秒消失 |
| 网络错误 | — | Toast 弹出："网络连接失败，请检查网络" |
| `SESSION_NOT_FOUND` | 404 | Toast 弹出："会话不存在或已被删除"，自动刷新会话列表 |

## 7. 状态管理

> React `useState` Hooks，列出组件需要管理的状态。

| State 变量 | 类型 | 初始值 | 用途 |
|------------|------|--------|------|
| `sessions` | `Session[]` | `[]` | 会话列表 |
| `currentSession` | `Session\|null` | `null` | 当前选中会话 |
| `records` | `QARecord[]` | `[]` | 当前会话的问答记录 |
| `query` | `string` | `""` | 输入框内容 |
| `loading` | `boolean` | `false` | 问答请求是否进行中 |
| `capabilities` | `Caps\|null` | `null` | 能力探测结果 |
| `error` | `string\|null` | `null` | 当前错误信息 |
| `showDeleteConfirm` | `string\|null` | `null` | 待删除的会话 ID，非 null 时显示确认弹窗 |
