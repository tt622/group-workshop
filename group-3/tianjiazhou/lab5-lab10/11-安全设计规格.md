## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至未授权的外部服务 | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 可追溯性 | 所有操作可通过 traceId 追踪，LLM 回答标注来源 | 投研合规要求 |

## 2. 认证与授权

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 当前教学版：无认证（开放访问）；生产版：JWT Token + 内部 SSO 对接 |
| Token 有效期 | 生产版：Access Token 30min, Refresh Token 7d |
| 密码策略 | 生产版：最少 8 位，含大小写 + 数字，90 天强制更换 |
| 多因素认证 | 生产版：管理员角色登录强制 MFA |

### 2.2 角色与权限矩阵

| 角色 | 研报上传 | 研报阅读 | 问答交互 | 观点分析 | 系统管理 |
|------|----------|----------|----------|----------|----------|
| 投研分析师 | ✓ | ✓ | ✓ | ✓ | ✗ |
| 基金经理 | ✗ | ✓ | ✓ | ✓ | ✗ |
| 研究助理 | ✗ | ✓ | ✓ | ✗ | ✗ |
| 系统管理员 | ✓ | ✓ | ✓ | ✓ | ✓ |

> **当前教学版**：不实施角色控制，所有用户拥有全部权限。

## 3. 数据安全

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/HTML 研报文件 | 本地存储，禁止外传，生产需加密存储 |
| 问答记录 | **内部** | 用户提问与 AI 回答 | 本地 JSON 存储，生产需加密 + 定期清理 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 教学版不采集；生产版脱敏展示 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |
| LLM API Key | **机密** | DASHSCOPE_API_KEY | 环境变量存储，禁止硬编码，禁止日志输出 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | 当前教学版 HTTP；生产版 TLS 1.2+ | 生产环境所有 API 通信强制 HTTPS |
| 存储加密 | 当前教学版明文 JSON；生产版 AES-256 | 研报文件与问答记录落盘加密 |
| 密钥管理 | 环境变量（`.env`） | 生产版迁移至 KMS，密钥定期轮转 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | query 长度 1~500，session_id 非空 UUID 格式 |
| SQL 注入防护 | 不使用 SQL（JSON 存储） | P1/P2 迁移数据库后需参数化查询 |
| XSS 防护 | 前端 React 自动转义 + CSP | 研报内容渲染时防止脚本注入 |
| CSRF 防护 | 当前教学版不实施；生产版 CSRF Token | 状态变更接口（POST/DELETE）需校验 |
| 速率限制 | 当前教学版无限制；生产版 100 req/min/user | 防止接口滥用和 DDoS |
| 文件上传校验 | 类型白名单（PDF/HTML）+ 大小限制（50MB） | 仅允许研报格式文件上传 |

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 优先使用内网百炼服务 / CoPaw 桥接，禁止调用未授权外网 API | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 与用户输入隔离，query 预处理过滤特殊指令 | 注入测试用例全部通过（TC-M01-060） |
| 模型幻觉 / 错误信息 | 回答标注来源 `answer_source`，观点整合标注研报出处 | 所有回答均携带来源标识 |
| 敏感信息泄露（模型回复） | 生产版增加输出过滤层，正则匹配敏感信息并脱敏 | 脱敏规则覆盖身份证、手机号、账号等 |

## 6. 审计与日志

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| API 请求 | traceId、端点、耗时、状态码 | 90 天 | 5xx 错误率 > 5% 告警 |
| LLM 调用 | 模型、耗时、降级事件 | 90 天 | 连续降级 > 10 次告警 |
| 研报访问 | 用户 ID、文档 ID、操作类型 | 365 天（生产版） | 异常批量下载告警 |
| 数据导出 | 导出内容摘要、目标 | 365 天（生产版） | 每次导出需审批 |
| 系统配置变更 | 变更人、变更前后值 | 永久（生产版） | 关键配置变更即时通知 |

> **当前教学版**：仅记录 Flask 标准日志（INFO 级别），不实施审计日志。

## 7. 安全威胁清单（STRIDE 简表）

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 |
|------|------|----------|----------|--------|
| **S** (Spoofing) | 伪造用户身份访问研报 | **高** | 生产版：强认证 + JWT Token 校验 | 后端开发 |
| **T** (Tampering) | 研报内容在传输中被篡改 | 中 | 生产版：TLS + 文件哈希校验 | 运维 |
| **I** (Info Disclosure) | 内部数据通过 AI 模型泄露 | **高** | 内网部署 + 输出过滤 | AI 工程师 |
| **D** (DoS) | 大量请求导致服务不可用 | 中 | 速率限制 + 三级降级策略 | 后端开发 |
| **E** (Elevation) | 普通用户获取管理员权限 | 高 | 生产版：RBAC 权限矩阵 | 后端开发 |
| **R** (Repudiation) | 用户否认操作行为 | 低 | 审计日志 + traceId 链路追踪 | 运维 |
