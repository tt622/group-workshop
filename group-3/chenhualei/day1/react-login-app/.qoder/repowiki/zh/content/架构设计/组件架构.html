简介

React Login App 采用组件化开发模式，将 UI 拆分为独立、可复用的组件。组件架构遵循 React 最佳实践，实现了清晰的职责分离和高效的代码复用。

组件分类

页面组件（Pages）

页面组件对应路由，负责整体页面布局和协调：

  • LoginPage: 登录页面，包含登录表单
  • DashboardPage: 仪表板页面，展示用户信息
  • TetrisPage: 俄罗斯方块游戏页面

业务组件（Components）

可复用的业务逻辑组件：

  • LoginForm: 登录表单组件，包含表单验证逻辑

路由组件（Routes）

路由相关的功能性组件：

  • ProtectedRoute: 受保护路由，实现权限控制

组件通信

Props 传递

父子组件通过 Props 进行数据传递：

// 父组件传递
<LoginForm onSubmit={handleLogin} />

// 子组件接收
function LoginForm({ onSubmit }) {
  // 使用 onSubmit
}

状态管理

跨组件状态通过 Zustand Store 管理：

// 定义 Store
const useAuthStore = create((set) => ({
  user: null,
  login: (user) => set({ user }),
}));

// 组件中使用
const user = useAuthStore((state) => state.user);

事件传递

子组件通过回调函数向父组件传递事件：

// 子组件触发事件
<button onClick={() => onSubmit(data)}>提交</button>

组件设计原则

单一职责

每个组件只负责一个功能领域，避免组件过于复杂。

可复用性

设计通用组件，通过 Props 配置实现不同场景的使用。

可测试性

组件逻辑清晰，便于编写单元测试。

命名规范

  • 组件名使用 PascalCase
  • 文件名与组件名一致
  • Props 使用 camelCase
