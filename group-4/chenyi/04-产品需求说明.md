## 1. 愿景与目标

> 把 `03` §1 的痛点翻译成正面目标。格式："构建 XXX，帮助用户在 XXX 内完成 XXX"。

**示例 — 痛点→目标映射**：

| 03 痛点 | 04 目标 |
|---------|---------|
| 信息分散、无统一入口 | 统一会话问答入口 |
| 复盘成本高 | 会话历史自动沉淀，随时回看 |

| 03 痛点 | 04 目标 |
|---------|---------|
| 研报阅读耗时，关键信息提取不完整 | 构建研报智能解析能力，帮助用户在 30 秒内获取评级、目标价、核心观点等结构化摘要 |
| 多份研报对比困难，依赖手动整理 | 内置多报对比功能，自动生成可导出的关键指标对比表，替代手工 Excel |
| 股票数据分散，需切换多个系统 | 在研报阅读界面直接检索相关股票数据，消除系统切换成本 |

## 2. 目标用户

> 把 `03` §2 的一句话展开成具体角色。思考：谁会用？用来做什么？多常用？

| 角色 | 使用目标 | 频率 |
|------|----------|------|
| 投研分析师 | 快速查询投研观点与市场解释 | 每天多次 |
| 基金经理 / 投研总监 | 快速获取研报核心结论与评级，辅助投资决策 | 每天 1~3 次 |
| IT 运维人员 | 维护本地部署环境，监控模型服务状态与降级切换 | 按需（异常时） |

## 3. 核心业务场景

> 把 `03` §2 提案概要拆成 4~5 条用户路径。每条场景 = 一条用户操作动线。

**示例 — SC-01**：

**SC-01 新建会话并提问**：用户点击"新会话" → 输入问题 → 获得回答（含来源标识）→ 回答自动落库

| 编号 | 场景名 | 用户路径 |
|------|--------|----------|
| SC-01 | 新建会话并提问 | （见上方示例） |
| SC-02 | 上传研报并解析 | 用户上传 PDF/HTML 研报 → 系统自动解析 → 提取标题、评级、目标价、核心观点 → 结构化摘要展示在会话界面 |
| SC-03 | 基于研报内容提问 | 用户在已解析研报的会话中输入自然语言问题 → 系统检索研报内容作答 → 回答附带原文引用段落 → 记录落库 |
| SC-04 | 多报关键指标对比 | 用户上传 2 份及以上研报 → 点击"生成对比表" → 系统在 30 秒内输出各券商评级、目标价、核心观点对比 → 支持导出 |
| SC-05 | 查看与管理历史会话 | 用户进入会话列表 → 浏览历史会话标题与时间 → 点击进入查看完整问答记录 → 可删除不需要的会话 |

## 4. 产品规则

> 把 `03` §4 成功标准 + §3 范围约束展开成可落地的规则。参考方向：输入约束、返回字段、状态管理、降级策略。

| 编号 | 规则 | 来源 |
|------|------|------|
| R-01 | 每条提问必须绑定 `session_id` | 03 §2 "会话式问答" |
| R-02 | `query` 长度 1～500 字符，为空或超长返回明确错误 | 03 §4 "P0 全部可测" |
| R-03 | 回答必须携带字段：`answer`、`answer_source`（`llm` / `local` / `demo`）、`cited_passages`（原文引用段落列表）、`session_id`、`created_at` | 03 §2 "引用原文作答 · 可追溯" |
| R-04 | 外部 LLM 不可用时自动降级：CoPaw → 百炼 → Demo 模式，界面需明确提示当前模式，不得返回空白页或未处理异常 | 03 §5 风险应对 |
| R-05 | 会话状态仅有"活跃"与"已删除"两种；删除为软删除，不物理清除数据；列表接口默认只返回活跃会话 | 03 §3 "会话管理：创建、列表、删除" |

## 5. 能力与优先级

> `03` §3 范围表写了"做什么"，04 按 P0/P1 排优先级。P0 = 必须实现，阻塞发布；P1 = 应当实现，不阻塞首版。

| 子能力 | 优先级 | 对应场景 |
|--------|--------|----------|
| 会话管理（列表 / 新建 / 删除） | **P0** | SC-01, SC-05 |
| 问答提交与三级降级 | **P0** | SC-01, SC-03 |
| 研报上传与结构化解析（PDF/HTML） | **P0** | SC-02 |
| 基于研报内容的自然语言问答与原文引用 | **P0** | SC-03 |
| 多报关键指标对比表生成与导出 | **P1** | SC-04 |

## 6. 验收与成功标准（PRD 层）

> 从 `03` §4 可度量标准出发，加上演示条件。
> **参考句式**："种子数据 ≥ N 条下，XXX 在 N 分钟内完成"

| 标准 | 度量 |
|------|------|
| 可演示 | 种子数据 ≥ 10 条下，主路径 + 历史 + 能力状态 **3 分钟内**演示完毕 |
| 离线可用 | 未配置密钥时系统仍返回离线演示回复（`answer_source=demo`） |
| 研报解析准确 | 以 10 份样例研报验收，评级、目标价、核心观点提取准确率 ≥ 90% |
| 多报对比时效 | 上传 2 份及以上研报后，对比表在 30 秒内生成完毕，字段完整且支持导出 |

## 7. 与 05 用户故事的关系

| 本文（PRD） | 作用 | 05 中落地 |
