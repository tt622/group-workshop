## 1. 背景与问题

> 描述用户遇到的问题，至少写 2~3 条痛点。参考：信息获取、历史记录、可追溯性、系统可用性。

| 痛点 | 来源 |
|------|------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 用户访谈 |
| 阅读耗时：每天需阅读 10+ 份研报，每份 30-50 页，平均耗时 2-3 小时，关键信息（评级、目标价、核心观点）提取不完整 | S-01 会议 / S-02 邮件 |
| 对比困难：不同券商研报格式各异，需手动复制粘贴至 Excel 整理，无法自动生成对比表 | S-02 邮件 / S-03 访谈 |

## 2. 提案概要

> 用 3~5 句话概括要做什么。参考格式："在 XXX 应用内交付 YYY 能力"。要素：会话式问答 · 能力降级 · 可追溯

在投研工作台内，构建一套本地部署的研报智能问答系统，支持 PDF/HTML 格式研报的自动解析与结构化提取（评级、目标价、核心观点），并提供基于研报内容的自然语言会话式问答，引用原文作答。系统内置多份研报关键指标一键对比功能，覆盖分析师从"读报"到"对比"的完整工作流。所有数据全程本地部署、不出域，满足内部合规要求；在外部 LLM 不可用时提供三级降级保障，确保核心功能持续可用；所有问答记录与来源可追溯，支持历史会话查阅。

## 3. 范围边界

> 明确"做什么"和"不做什么"，避免范围蔓延。各写 3~4 条。

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 会话管理：创建、列表、删除 | 多租户/行级数据权限 |
| 研报解析：支持 PDF/HTML，自动提取标题、评级、目标价、核心观点 | 舆情监控与关键词实时告警（列入后续版本） |
| 自然语言问答：基于研报内容回答，支持引用原文段落 | 研报自动推送与订阅通知机制 |
| 研报对比：多份研报关键指标自动生成可导出的对比表 | 对接外部云端 LLM API（仅支持本地部署模型） |

## 4. 成功标准（可度量）

> 写出 3~4 条可量化的验收标准。句式：用户在 X 分钟内完成 XXX 全链路 / 未配置外部模型时返回 XXX

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 3 分钟内完成"新建会话 → 提问 → 查看历史"全链路 |
| 关键信息提取准确 | 评级、目标价、核心观点提取准确率 ≥ 90%（以 10 份测试研报人工对照验收） |
| 多报对比可用 | 上传 2 份及以上研报后，系统在 30 秒内自动生成可导出的对比表，字段完整 |
| 降级保障有效 | 外部 LLM 不可用时，系统自动切换至本地模型并在界面给出明确提示，不返回空白页或未处理异常 |

## 5. 风险（简表）

> 列出 2~3 个主要风险。常见方向：外部依赖不可用、数据存储并发、模块间契约延期。

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| PDF 格式多样，解析失败率高 | 关键信息提取准确率不达标 | 引入 OCR 兜底层；预发布阶段以 20 份真实研报验收解析效果 |
| 本地模型推理速度慢 | 用户等待时间过长，体验下降 | 设置 30s 超时提示 + 流式输出；优先评估 4-bit 量化模型 |

## 6. 立项前置条件（Go / No-Go）

- ☐ 后端与前端可独立启动
- ☐ 演示数据目录可读写
- ☐ 本地 LLM 或 Demo 模式可正常返回问答结果，无阻断性错误
- ☐ IT 部完成安全评估，确认本地部署方案符合数据合规要求（截止 2026-04-20，负责人：张产品）
