# 手写版

## 1. 背景与问题

投研条线的用户在研报阅读上存在以下问题：

1. 痛点：阅读耗时长 | 原因：研报数据过多，人类阅读速度较慢 | 来源：用户访谈
2. 痛点：信息提取不完整 | 原因：研报结构或格式都存在差异，存在信息无法完全对称的情况 | 来源：用户访谈
3. 痛点：研报对比困难 | 原因：人类有体力上限，在投入和产出间存在产能和需求的错配 | 来源：用户访谈
4. 痛点：股票数据查询不便 | 原因：数据在多个平台和系统间分散，来回切换麻烦 | 来源：用户访谈


## 2. 提案概要

要求：建立研报分析平台，解决痛点问题，同时要确保系统稳定运行，可适度添加定制化需求（每个研究员关注的重点信息不同）。用户必须以问答形式使用整个功能的核心过程。


## 3. 范围边界

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 面向经理和研究员 | 平台不开放给非投研条线及无权限人员 |
| 在明确用户需要的前提下，问答方式完成研报分析与关键信息提取与统计 | 不要在未清除用户要求的情况下进行工作 |
| 整合多个来源的，最新的研报、新闻、公告等信息 | 不要根据过少的来源就给出投研判断 |

## 4. 成功标准（可度量）

| 标准 | 度量方式 |
|------|----------|
| 用户能够通过问答的形式快速阅读研报 | 用户在 3 分钟内完成"新建会话 → 提问 → 查看历史"全链路 |
| 系统稳定运行 | 全年可用时间达到4个9 |
| 不因单一数据来源故障影响整体系统运行 | 故障后能降级运行，满足80%的使用要求，且不影响其他核心系统链路 |

## 5. 风险（简表）

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| 数据存储超出磁盘需要 | 系统无法继续工作 | 定期归档文档 |
| 功能整体故障 | 影响投资交易系统运行 | 不能对投研核心链路使用的功能强依赖 | 

## 6. 立项前置条件（Go / No-Go）

- ☐ 后端与前端可独立启动
- ☐ 演示数据目录可读写
- ☐ 用户需求明确
