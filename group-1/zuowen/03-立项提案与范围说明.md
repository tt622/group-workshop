## 1. 背景与问题

> 描述用户遇到的问题，至少写 2~3 条痛点。参考：信息获取、历史记录、可追溯性、系统可用性。

| 痛点 | 来源 |
|------|------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 用户访谈 |
| 对比效率低：多向调研来源的研报格式不统一，无法快速横向对比 | 邮件沟通 |

## 2. 提案概要

> 用 3~5 句话概括要做什么。参考格式："在 XXX 应用内交付 YYY 能力"。要素：会话式问答 · 能力降级 · 可追溯

在投研问答助手应用内交付会话式问答能力。支持用户通过自然语言提问，快速获取多来源研报摘要与对比分析，支持股票数据搜索，用户会话要支持管理。系统支持跨平台数据源接入，输出用户友好的可视化对比表格。当外部 LLM 不可用时，采用三级降级策略（CoPaw → 百炼 → Demo 模式）保障服务可用性。

## 3. 范围边界

> 明确"做什么"和"不做什么"，避免范围蔓延。各写 3~4 条。

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 会话管理：创建、列表、删除 | 多租户/行级数据权限 |
| 支持多种数据源接入（研报、新闻、数据库） | 本地整理的研报不支持上传 |
| 支持分析对比，输出可视化对比表格 | 用户自定义指标/关键词不支持 |

## 4. 成功标准（可度量）

> 写出 3~4 条可量化的验收标准。句式：用户在 X 分钟内完成 XXX 全链路 / 未配置外部模型时返回 XXX

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 3 分钟内完成"新建会话 → 提问 → 查看历史"全链路 |
| 生成关键摘要 | 用户输入公司名，生成关键信息摘要，汇总摘要返回 |
| 对比查询可用 | 用户输入公司名、指标，生成关键信息摘要，输出用户友好的可视化对比表格 |
| 降级机制生效 | 外部 LLM 不可用时，系统自动切换至 CoPaw/百炼/Demo 模式 |

## 5. 风险（简表）

> 列出 2~3 个主要风险。常见方向：外部依赖不可用、数据存储并发、模块间契约延期。

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| 数据源接入不稳定 | 对比分析结果不完整 | 预设缓存机制，支持离线模式 |
| 并发访问性能瓶颈 | 用户体验下降 | 采用异步处理，流式输出 |
| 权限控制 | 数据泄露 | 强身份校验，无权限拒绝回答 |

## 6. 立项前置条件（Go / No-Go）

- ☐ 后端与前端可独立启动
- ☐ 演示数据目录可读写
- ☐ LLM API 密钥已配置（CoPaw/百炼至少一个可用）
- ☐ 研报数据源接入测试通过
