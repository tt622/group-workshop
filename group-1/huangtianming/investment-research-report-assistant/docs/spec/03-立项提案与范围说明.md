## 1. 背景与问题

> 描述用户遇到的问题，至少写 2~3 条痛点。参考：信息获取、历史记录、可追溯性、系统可用性。

| 痛点 | 来源 |
|------|------|
| 信息分散：研报、新闻、数据分布在多个系统，无统一问答入口 | 用户访谈 |
| 研报阅读耗时：每日阅读10+份研报，耗时超过2小时，难以获取关键信息 | 会议/邮件 |
| 关键信息提取不完整：研报格式、样式各异，关键信息分散，人工难以提取关键信息| 会议/邮件 |
| 多份研报比对困难：缺乏进行比对的平台和工具，依赖人工使用复制粘贴形式对多份同一主题或标的研报的比对| 会议/邮件 |
| 股票信息分散：查询股票信息需要在多个系统进行切换，无法在研报阅读页面直接查询关联股票讯息| 访谈 |

## 2. 提案概要

> 在投研工作台，构建一套本地智能研报问答系统，支持 WORD/PPT/PDF/HTML 格式研报的自动解析与结构化提取（评级、目标价、核心观点），并提供基于研报内容的自然语言问答，引用原文作答。同时内置多份研报关键指标一键对比功能，以及相关股票数据直接检索入口，覆盖分析师从"读报"到"对比"到"查数据"的完整工作流。系统须全程本地部署、数据不出域，满足内部数据安全合规要求；并在外部 LLM 不可用时提供三级降级保障，确保核心功能持续可用。

## 4. 成功标准（可度量）

> 写出 3~4 条可量化的验收标准。句式：用户在 X 分钟内完成 XXX 全链路 / 未配置外部模型时返回 XXX

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 3 分钟内完成"新建会话 → 提问 → 查看历史"全链路 |
| 研报关键信息提取准确 | 评级、目标价、核心观点提取准确率 ≥ 90%（以 20 份测试研报人工对照验收） |
| 多研报对比可用 | 上传两份研报后，系统可以在2分钟内生成对比列表 |
| 相关行情可查 | 推荐查询在研报中出现的股票代码，可查询全市场股票信息 |

## 5. 风险（简表）

> 列出 2~3 个主要风险。常见方向：外部依赖不可用、数据存储并发、模块间契约延期。

| 风险 | 影响 | 应对 |
|------|------|------|
| 外部 LLM 不可用 | 问答功能降级 | 三级降级：CoPaw → 百炼 → Demo 模式兜底 |
| PDF 格式多样，解析失败率高 | 提取准确率不达标 | 引入 OCR 兜底层 |
