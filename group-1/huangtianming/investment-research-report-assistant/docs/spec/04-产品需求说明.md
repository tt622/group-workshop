> **04 的定位**：`03` 说了"为什么做"，04 翻译成"做什么、不做什么、多重要"。`05` 再拆成用户故事和验收标准。

## 1. 愿景与目标

**愿景**：构建一套本地化的投研研报智能助手，帮助分析师在 3 分钟内完成"新建会话 → 导入研报 → 提问 → 对比 → 查数据"全链路，将单份研报的关键信息抓取从 30 分钟压缩到 2 分钟内。

**痛点 → 目标映射**：

| 03 痛点 | 04 目标 |
|---------|---------|
| 信息分散、无统一问答入口 | 以会话为中心的统一研报问答工作台 |
| 研报阅读耗时 | 一键解析 + 结构化摘要卡片，直达关键信息 |
| 关键信息提取不完整 | 评级/目标价/核心观点结构化提取，准确率 ≥ 90% |
| 多份研报对比困难 | 2 分钟内生成多研报关键指标对比结果 |
| 股票信息分散 | 研报上下文直达股票基础与行情信息 |

## 2. 目标用户

| 角色 | 使用目标 | 频率 |
|------|----------|------|
| 投研分析师 | 快速读懂研报、提炼观点、比对结论 | 每日多次（10+ 份研报） |
| 高级分析师 / 组长 | 审阅下属研究、比对多方观点、生成汇报 | 每日数次 |
| 投资经理 | 基于研报与行情快速决策 | 每日数次 |

## 3. 核心业务场景

| 编号 | 场景名 | 用户路径 |
|------|--------|----------|
| SC-01 | 新建会话并管理上下文 | 点击"新会话" → 自动切换到空会话 → 可切换/删除历史会话 |
| SC-02 | 导入并解析研报 | 在会话中上传 WORD/PPT/PDF/HTML → 系统自动解析 → 展示解析状态与失败原因 |
| SC-03 | 查看结构化摘要 | 解析完成 → 摘要卡片展示评级/目标价/核心观点 → 点击字段回溯原文 |
| SC-04 | 基于研报提问 | 输入自然语言问题 → 系统返回答案 + 原文引用 → 记录入会话历史 |
| SC-05 | 多研报对比 | 勾选 ≥ 2 份研报 → 点击"对比" → 2 分钟内生成关键指标对比表 |
| SC-06 | 关联股票查询 | 从研报上下文点击股票标签 → 右侧面板展示基础/行情信息，无需离开当前页面 |
| SC-07 | 外部模型异常兜底 | LLM 不可用时 → 返回明确提示或降级结果，保留当前数据 |

## 4. 产品规则

| 编号 | 规则 | 来源 |
|------|------|------|
| R-01 | 每次问答 MUST 绑定 `session_id`，跨会话数据不得串线 | 03 §3 会话管理 |
| R-02 | `query` 长度 1–500 字符，为空或超长返回明确错误 | 03 §4 主路径可走通 |
| R-03 | 回答 MUST 携带至少一处原文引用标识；缺证据时明确"未在当前研报中找到依据"，不得伪造 | 03 §2 提案概要 |
| R-04 | 支持的研报格式：`WORD / PPT / PDF / HTML`；单份解析失败不得影响其他研报 | 03 §3 研报解析 |
| R-05 | 结构化字段固定为：`rating`（评级）、`target_price`（目标价）、`core_views`（核心观点）；无法识别时标记"未提取到" | 03 §4 提取准确率 |
| R-06 | 多研报对比 MUST 保留每个字段的来源研报标识，不得用其他研报内容代填 | 03 §2 对比能力 |
| R-07 | 股票查询 MUST NOT 要求用户切换到外部系统 | 03 §4 相关行情可查 |
| R-08 | 外部 LLM 异常时 MUST NOT 暴露密钥 / 内部 URL / 调用栈 | 03 §5 风险 |
| R-09 | 所有用户数据、研报内容 MUST 本地存储，数据不出域 | 03 §2 提案概要 |

## 5. 能力与优先级

| 子能力 | 优先级 | 对应场景 |
|--------|--------|----------|
| 会话管理（列表 / 新建 / 删除 / 切换） | **P0** | SC-01 |
| 多格式研报解析（WORD/PPT/PDF/HTML + OCR 兜底） | **P0** | SC-02 |
| 结构化字段提取（评级/目标价/核心观点） | **P0** | SC-03 |
| 基于研报内容的问答 + 原文引用 | **P0** | SC-04 |
| 多研报关键指标对比 | **P0** | SC-05 |
| 关联股票信息查询 | **P0** | SC-06 |
| 外部模型异常的降级 / 友好兜底 | **P1** | SC-07 |
| 研报舆情监控 / 自动订阅 | P2 | 后续版本 |

## 6. 验收与成功标准（PRD 层）

| 标准 | 度量 |
|------|------|
| 主路径可走通 | 种子数据 ≥ 3 份研报下，用户 3 分钟内完成"新建会话 → 导入 → 提问 → 查看历史"全链路 |
| 提取准确率 | 20 份测试研报下，评级/目标价/核心观点综合提取准确率 ≥ 90%（人工对照） |
| 对比及时性 | 上传两份研报后 2 分钟内生成可查看的对比列表 |
| 股票联动 | 研报中识别出的股票代码可在当前界面一键查询到基础/行情信息 |
| 异常兜底 | 外部模型异常时系统不崩溃、不泄露敏感细节，当前会话与已上传研报保留 |
