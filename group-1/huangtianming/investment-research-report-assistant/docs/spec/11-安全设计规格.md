## 1. 安全目标与合规要求

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 研报原文、用户问答、会话数据禁止传输至外部（LLM 除外，且走内网桥接优先） | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 可追溯 | 所有问答结果必须能溯源到研报原文 | 投研合规要求 |
| 外部调用可脱敏 | 错误信息不得回传密钥/URL/调用栈 | `05` AC-007-03 |

## 2. 认证与授权

### 2.1 认证机制（教学版 vs 生产版）

| 项目 | 教学版（当前） | 生产版（规划） |
|------|---------------|----------------|
| 认证方式 | 无（开放访问） | 内部 SSO / JWT |
| Token 有效期 | — | Access 30min, Refresh 7d |
| 密码策略 | — | 最少 8 位，含大小写+数字，90 天轮换 |
| 多因素认证 | — | 敏感操作触发 MFA |

### 2.2 角色与权限矩阵（生产版）

| 角色 | 研报上传 | 研报阅读 | 问答交互 | 对比分析 | 股票查询 | 系统管理 |
|------|----------|----------|----------|----------|----------|----------|
| 分析师 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 高级分析师 / 组长 | ✓ | ✓ | ✓ | ✓ | ✓ | ✗ |
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 只读访客 | ✗ | ✓ | ✗ | ✗ | ✓ | ✗ |

## 3. 数据安全

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | WORD/PPT/PDF/HTML 文件 | 本地存储，访问审计，生产版加密 |
| 问答记录 | **内部** | 用户提问 + AI 回答 + 原文引用 | 本地 JSON，定期清理策略 |
| 结构化摘要 | **内部** | 评级/目标价/核心观点 | 与研报同等保护级别 |
| 对比结果 | **内部** | 多研报聚合表 | 同上 |
| 股票查询结果 | 一般 | 公开行情数据 | 无额外要求 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 脱敏展示，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 教学版 | 生产版 |
|------|--------|--------|
| 传输加密 | HTTP（localhost） | 强制 TLS 1.2+ |
| 存储加密 | 明文 JSON + 本地文件 | AES-256 落盘加密 |
| 密钥管理 | 环境变量 `.env` | KMS / Vault |

## 4. 接口安全

| 防护措施 | 实现方式 | 说明 |
|----------|----------|------|
| 输入校验 | 服务端白名单校验 | 所有字段类型/长度/格式严格校验（见 `09` §17） |
| SQL 注入防护 | JSON 存储不涉及 SQL；生产版禁止字符串拼接 SQL | — |
| XSS 防护 | 前端对研报片段 / 回答使用 React 默认转义 + CSP | 禁止 `dangerouslySetInnerHTML` 未过滤使用 |
| CSRF 防护 | 生产版：状态变更接口需 CSRF Token | 教学版暂缺 |
| 速率限制 | 生产版：100 req/min/user，ask 端点 30 req/min | 教学版不限速 |
| 文件上传校验 | 格式白名单 + 大小限制 + 魔数校验 | 见 `07` §5 |
| 路径穿越防护 | 上传文件名 UUID 重命名，禁止使用原始文件名写盘 | 防御路径穿越 |

## 5. LLM / AI 安全

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 优先 CoPaw（内网桥接），禁止将完整研报原文直发公网 LLM | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤，对研报内容转义 | 专项 TC（`13`） |
| 模型幻觉 / 伪造引用 | 所有回答必须附 `citations[]`，无证据时明确"未找到依据" | AC-004-02 / 03 |
| 敏感信息泄露（回复） | 输出过滤层匹配敏感词并脱敏 | 专项 TC |
| 外部错误信息泄露 | 统一捕获异常，返回通用 `UPSTREAM_ERROR`；不得回传调用栈/URL/Key | AC-007-03 |

## 6. 审计与日志

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 登录 / 登出（生产版） | 用户 ID、时间、IP、结果 | 180 天 | 连续失败 ≥ 5 次告警 |
| 研报上传 / 访问 | 用户 ID、report_id、操作类型、traceId | 365 天 | 异常批量下载告警 |
| 问答调用 | 用户 ID、session_id、answer_source、response_time_ms | 180 天 | 错误率告警 |
| 对比生成 | compare_id、report_ids、elapsed_ms | 180 天 | 超过 SLO 告警 |
| 外部 LLM 调用 | provider、model、状态、耗时（不含内容） | 90 天 | 失败率告警 |
| 数据导出（生产版） | 导出摘要、目标 | 365 天 | 每次导出需审批 |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 即时通知 |

## 7. 安全威胁清单（STRIDE）

| 类型 | 威胁 | 风险等级 | 应对措施 |
|------|------|----------|----------|
| **S** Spoofing | 伪造用户身份访问研报 | 高 | 强认证 + Token 校验（生产版） |
| **T** Tampering | 研报内容在传输中被篡改 | 中 | TLS + 文件哈希校验 |
| **R** Repudiation | 用户否认上传/问答行为 | 中 | 审计日志 + traceId 全链路 |
| **I** Information Disclosure | 内部数据通过 AI 模型泄露 | **高** | 内网模型优先 + 输出过滤 + 错误脱敏 |
| **D** Denial of Service | 大量请求/超大文件导致服务不可用 | 中 | 速率限制 + 文件大小限制 + 降级策略 |
| **E** Elevation of Privilege | 通过 path traversal 读写系统文件 | 中 | 上传文件重命名 + 路径白名单 |

## 8. 部署安全

- [ ] 生产 / 开发环境严格隔离，数据不互通
- [ ] 数据库与文件存储仅限应用服务器 IP 白名单
- [ ] 敏感配置（API Key、模型 URL）通过环境变量注入，不入代码仓库
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] `.env`、`data/` 目录加入 `.gitignore`，禁止提交
- [ ] 上传目录独立 mount，禁止 `exec` 权限
- [ ] 生产版开启 WAF + 日志审计
