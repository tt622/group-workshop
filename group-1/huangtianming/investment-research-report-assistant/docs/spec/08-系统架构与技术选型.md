## 1. 架构总览

### 1.2 架构风格

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 教学清晰度：前端独立开发服务器 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度 |
| **分层架构** | Route → Service（Agent/Parser/Extractor/Stock） → Storage | 替换存储引擎/外部模型时上层零改动 |
| **领域服务拆分** | ReportParser / StructuredExtractor / CoPawAgent / StockService | 单一职责，每个 Story 对应一条主干服务链 |
| **数据本地化** | JSON + 本地文件系统 | 满足 R-09"数据不出域" |

## 2. 后端分层结构

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `agent_bp.py` | 参数校验、HTTP 状态码、traceId 注入 | 直接读写文件、调用 LLM |
| **Agent** | `agent.py` | 三级降级编排、结果组装、引用拼装 | 感知 HTTP request/response |
| **Parser** | `parsers/word.py / ppt.py / pdf.py / html.py / ocr.py` | 文件解析、文本抽取 | 调用 LLM、写入存储 |
| **Extractor** | `extractor.py` | 结构化字段抽取（评级/目标价/核心观点）、LLM prompt 编排 | 感知路由 |
| **Stock** | `stock_service.py` | 外部行情数据 HTTP 调用、数据消歧 | 操作本地存储 |
| **Provider** | `copaw_bridge.py` / `bailian_qa.py` | HTTP 调用外部 LLM API | 操作本地存储 |
| **Storage** | `storage.py` | 文件 CRUD、数据校验、级联删除 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）≥ 18 | 教学通用性最高 |
| 构建工具 | Vite ≥ 5 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` | 状态数量可控，不引入 Redux/Zustand |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| 样式方案 | 原生 CSS + CSS 变量 | 教学可读性，避免 Tailwind/Styled 认知成本 |
| 代码组织 | 按页面区块拆分：`Sidebar / ReportsPane / SummaryPane / QAPane / ComparePane / StockPane` | 与 `06` 布局一一对应 |

## 4. 三级降级编排

```
ask(query, session_id, report_ids)
  ├─[1] CoPaw  → {source:"copaw",  llm:true}
  ├─[2] 百炼   → {source:"bailian",llm:true}
  └─[3] Demo   → {source:"demo",   llm:false}
```

| 提供商 | 文件 | 配置检测 | 超时 | 错误处理 |
|--------|------|----------|------|----------|
| **CoPaw** | `copaw_bridge.py` | `IRA_COPAW_*_URL` 非空 | 20 s | 返回 None 静默降级 |
| **百炼** | `bailian_qa.py` | `DASHSCOPE_API_KEY` 非空 | 120 s | 区分多类错误码 |
| **Demo** | `agent.py` | 始终可用 | 0 s | 模板拼接 + 空引用 |

**降级不跳级**：Agent 按列表顺序 `try_copaw → try_bailian → fallback_demo`，每级返回 None 时继续下一级。

## 5. 研报解析编排

```
POST /sessions/<id>/reports
  → ReportParser.detect_format(file)
  → 分发到对应 parser：
      .doc/.docx → python-docx
      .ppt/.pptx → python-pptx
      .pdf       → pdfplumber（文本层缺失 → OCR 兜底 rapidocr）
      .html/.htm → BeautifulSoup
  → 结果持久化到 Storage.add_report
  → 后台异步触发 StructuredExtractor.run
```

| 阶段 | 失败处理 |
|------|----------|
| 格式检测 | 415 `UNSUPPORTED_FORMAT` |
| 解析 | 422 `REPORT_PARSE_FAILED`，其他研报不受影响（AC-002-03） |
| 结构化提取 | 字段级 `null` / 空数组，前端展示"未提取到"（AC-003-01） |

## 6. 多研报对比编排

```
POST /sessions/<id>/compare { report_ids: [...] }
  → 并行读取各 report 的 summary
  → 按字段聚合：rating / target_price / core_views
  → 保留 report_id 来源（AC-005-02）
  → 返回 compareResult + elapsed_ms
```

## 8. 数据存储架构

### 8.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | JSON 文件 + 本地文件系统 | — | 适合教学、满足数据不出域 |
| P1 | SQLite + 本地文件系统 | 仅替换 Storage 层 | 支持并发写入 |
| P2 | PostgreSQL + 对象存储 | Storage + 部署架构 | 生产级 |

### 8.2 数据目录布局

```
backend/data/
├── sessions.json         # 会话元数据
├── qa_records.json       # 问答记录
├── reports.json          # 研报元数据 + 结构化摘要
├── compare_results.json  # 对比结果
└── files/
    └── {report_id}.{ext} # 研报原文件
```

## 10. ADR（架构决策记录）

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线低，无需 async/await |
| ADR-002 | JSON 文件而非 SQLite（P0 阶段） | 教学可见性高，学员易查看数据 |
| ADR-003 | 前端不拆组件库（原生 CSS） | 避免引入 UI 库认知负担 |
| ADR-004 | 研报解析与提取解耦（Parser ≠ Extractor） | Parser 只做文本层，Extractor 调用 LLM，符合单一职责 |
| ADR-005 | OCR 仅作为 PDF 兜底（非默认） | 90% PDF 有文本层，默认走快路径 |
| ADR-006 | 股票查询由后端代理而非前端直连 | 避免外部 API Key 暴露到浏览器 |

## 11. 安全架构

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT / 内部 SSO |
| CORS | 全开放 | 白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 研报文件 | 本地文件系统 | 加密存储 + 访问审计 |
| 外部 LLM 调用 | 优先 CoPaw（内网桥接） | 本地化模型部署 |
| 错误信息脱敏 | 固定错误码 + 通用文案 | 统一脱敏中间件 |
