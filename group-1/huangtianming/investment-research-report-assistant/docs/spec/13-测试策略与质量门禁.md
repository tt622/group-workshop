## 一、测试分层

| 层级 | 工具 | 覆盖对象 | 说明 |
|------|------|----------|------|
| **L1 Unit** | pytest | 单个函数/类 | 不依赖网络 |
| **L2 Integration** | pytest + Flask test_client | API 端点 + 存储 | 模拟 HTTP 请求 |
| **L3 Contract** | pytest + schema 断言 | 响应体结构与 `09` 一致 | 验字段名、类型 |
| **L4 E2E** | Playwright | 完整用户旅程 | 浏览器级操作 |

## 二、用例拆分原则

| 来源 | 计数方法 | 示例 |
|------|----------|------|
| 正常路径 | 每个端点的 happy path = 1 条 | `POST /ask → 200` |
| 错误码 | 每个 distinct `error.code` = 1 条 | `EMPTY_QUERY`、`UNSUPPORTED_FORMAT` |
| 分支路径 | 每个 if/else 降级分支 = 1 条 | 无 API Key → demo |
| 类型约束 | `09` 约定的字段类型 = 1 条 | `llm_used` 必须 bool |

## 四、质量门禁

| Gate-ID | 条件 | 阻塞合并 |
|---------|------|----------|
| **G-LINT** | `black --check` + `flake8` 通过 | 是 |
| **G-UNIT** | Unit 测试全绿 | 是 |
| **G-INT** | Integration 测试全绿 | 是 |
| **G-AC** | 全部 P0 AC 已覆盖 | 是 |
| **G-COV** | 后端覆盖率 ≥ 80% | 是 |
| **G-E2E** | P0 E2E 旅程全绿 | 是 |
| **G-PERF** | `response_time_ms < 5000` & `compare elapsed_ms < 120000` | 否（warning） |
| **G-SEC** | 错误响应不含密钥/URL/调用栈 | 是 |

### 执行命令

```bash
# 后端测试 + 覆盖率
cd backend
python -m pytest tests/ -q --cov=. --cov-report=term-missing

# 前端 E2E
cd frontend
npx playwright test
```

## 五、TDD 适用场景

| 场景 | TDD 价值 | 建议 |
|------|----------|------|
| 会话 CRUD | 低 | 写完补测试 |
| 参数校验 / 错误码 | 中 | 建议 TDD |
| 研报解析分发器 | 中 | 建议 TDD |
| 结构化提取 prompt | 高 | 强烈建议 TDD |
| 三级降级编排 | **高** | **必须 TDD** |
| 对比字段聚合 | **高** | **必须 TDD** |
